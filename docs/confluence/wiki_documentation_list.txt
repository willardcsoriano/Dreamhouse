and references that need to be authored in our corporate Confluence Wiki.

[ ] Complete Salesforce and Atlassian CLI Reference

2. RECOMMENDED ENGINEERING SOPS (PROPOSED ADDITIONS)
--------------------------------------------------------------------------------

[ ] Salesforce Scratch Org & Sandbox Lifecycle Management SOP
    - Standardized process for spinning up, configuring, and deleting scratch orgs.
    - Sandbox refresh schedule and environment promotion strategy (Dev -> QA -> UAT -> Prod).
    - Tracking local changes using source tracking (`sf project deploy start` / `sf project retrieve start`).

[ ] Local Developer Environment Setup & Onboarding Guide
    - Step-by-step onboarding checklist for new engineers.
    - Installing prerequisites (Node.js, Python, Salesforce CLI, IDE Extensions, Prettier, ESLint).
    - Setting up Git hooks (Husky, Lint-Staged) to ensure linting and formatting on commit.

[ ] Pull Request (PR) & Code Review Guidelines
    - Code review standards (performance, scalability, readability).
    - QA validation steps required before marking a PR as ready for review.
    - Checklist for reviewing Apex code (bulkification, limit management, trigger frameworks).

[ ] CI/CD Pipeline & Deployment SOP
    - Overview of the automated build and deployment pipelines (e.g., GitHub Actions, GitLab CI).
    - Package validation vs. deployment steps.
    - How to handle manual pre-deployment and post-deployment steps in Salesforce orgs.
    - Executing and tracking destructive changes (deleting metadata safely).

[ ] Test Data Management & Seeding SOP
    - Standardized scripts/commands for importing mock data into newly created scratch orgs (e.g., `sf data import tree`).
    - Guidelines on masking production data if used for sandbox testing.
    - Maintaining mock data JSON/CSV files in the repository.

[ ] Secrets, Credentials, & Security SOP
    - Rules on handling sensitive API keys, OAuth tokens, client secrets, and passwords.
    - Approved methods for managing environment variables (e.g., using secure credential stores, sfdx auth files).
    - Guidelines on running vulnerability scanners and static code analysis (e.g., PMD, Salesforce Scanner).

[ ] Atlassian Jira-GitHub/GitLab Integration SOP
    - How to use smart commits to reference Jira tickets directly from commit messages (e.g., "PROJ-123 #comment Resolved login issue").
    - Automatic ticket transition rules (e.g., moving a ticket to 'In Review' when a PR is opened).
